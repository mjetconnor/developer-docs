Puts to Idea Entities

When the target of a PUT request is an entity of type Idea, the body of the request may contain parameters as described here.

PUT parameter

Datatype

Version

Description

stage

Stage

3.8.2 and later

 Graduate the idea by changing it's stage.
See "Graduate the Idea" below.

subject

Concept

3.8.2 and later

 Move the idea to another category, possibly in another inquiry.
See "Move the Idea" below.

inquiry

Inquiry

3.8.2 and later

See "Move the Idea" below.

See Request Parameter Formats in the REST API reference section for a description of how entity values should be formatted as PUT request parameters.

 

Graduate the Idea

An idea may be graduated, or possibly regressed, by setting its "stage" property to another value.  Stage values are represented by entities of type "Stage".

The stage must be one that is defined in the idea's inquiry, that is, in the Spigit community or challenge in which the idea resides.  The list of applicable stages can be found in the governor associated with the idea or its inquiry.  See the diagram in the Idea Concepts topic that illustrates the relationship of ideas and inquiries to governors and stages.  For example, if you have the entity ID for an idea or inquiry, you can get the list of relevant stages with this REST query:

GET inquiryEid /governor/lifeCycle/stages

 

Move the Idea

An idea may be moved to another inquiry, another category in the same inquiry, or a category in a different inquiry, by setting one or both of its "subject" or "inquiry" properties.  An idea's "subject" property corresponds to its Spigit category.  An idea's "inquiry" property corresponds to its Spigit community or challenge.

If a value is given for the "subject" property in a PUT request, it must be an entity of type "Concept" that represents a category in some Spigit community or challenge.  It does not have to be in the same inquiry, that is, community or challenge, as the idea's current home.  If the category is in a different inquiry, the idea will be moved to that other inquiry as well as having its category changed.

The list of categories defined for an inquiry can be found in that inquiry entity's  "concepts" property.

If both "subject" and "inquiry" properties are given in a PUT request to an idea, the category specified for the "subject" value must be defined in the given "inquiry".

Some Spigit inquiries do not define any categories.  In this case, the idea may be moved to that inquiry by specifying the "inquiry" property but not the "subject" property.
