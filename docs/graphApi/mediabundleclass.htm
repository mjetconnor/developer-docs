MediaBundle Class

A bundle of related media resources.

 

Property

Datatype

Req

Description

parts

PlainObject

MUST

Object that contains the constituent resources of the bundle.  All properties of this object must have values that are MediaResources.  The property names are determined by usage but not by the object class.

object

Resource

MAY

If the bundle is in support of a particular thing, that thing is indicated by the value of this property.

purpose

Thing

MAY

Indicates the purpose of the bundle.

 

Purpose

Standard values for the purpose property:

Value

Description

"variants"

MediaBundles that have this purpose contain resources that differ only in format or scale.  The content should be the same in all member resources.

 

Parts

Generally speaking, the value of the "parts" property is an object with no restriction on what properties may or may not exist.  But regardless of the property names, their values must all be MediaResources.  One could scan all of the resources in the parts object to find the part that best matches a particular usage.

 

The bundle's "purpose" value is a cue to what property names may be expected.  When the bundle's purpose is "variants" (the only case defined so far), the property names SHOULD correspond to the "scale mnemonics" defined below.

* "Approx Size in Inches" values assume a device resolution on the order of 75 DPI.


As a general rule, images may be resized downward as much as 50%, but should not be resized upward because of potential display artifacts.  Consequently, a good practice is to generate images that are slightly larger in pixels compared to the expectations for a given mnemonic.
