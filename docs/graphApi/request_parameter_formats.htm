Request Parameter Formats

PUT and POST requests have message bodies that contain the parameters of those requests, as name/value pairs.  This topic describes the formats that are recognized for parameter values.

 

Datatype

Description

String

 When a string value is expected, any property escaped text value may be given.

Boolean

 These are recognizable boolean values:

true "true", "1"
false "false", "0"
 

Entity

Entities are represented by the URI that is their "entity ID".  When an entity value is expected, the full URI value may be given.

The value may instead be abbreviated to just its entity key part.  This is the part of the full URI value that is after the last slash character.

Examples:

A full entity URI:  "https://www.example.spigit.com/api/ds1/node-1423"

Abbreviated as just the entity key:  "node-1423"
