The syntax for handling POST functions in the REST API is described here.  PUT functions in the REST API is described here. The same rules apply for all posts:

  1. Posts are HTTP queries that use the POST or PUT method.
  2. The requests are sent to a "target" entity.  In other words, the URL of the request is the entity ID of a target entity.
  3. The body of the post request contains name-value pairs of information that are essentially arguments to the post.
  4. The post body may be a single object in JSON format whose properties are the post arguments.  Or it may be in form-urlencoded format with the form fields corresponding to the post arguments.
  5. Posts have result values that are returned in the response to the post request.  These are represented in the standard SpigitGraph REST response format.

The behavior of a post is determined by the type of the target entity and two key properties in the post body - action and type.  The default value for action is "post".  For "post" actions, the value of type indicates the type of thing you are trying to post.

Posts are very much like method calls in object orientation.  The target entity is just like the target object of methods.  The action value is the method name.  The other post body values are the arguments to the method.  The response contains the result of executing the method.
