Puts to User Entities

When the target of a PUT request is an entity of type User,  the body of the request may contain parameters as described here.

 

PUT parameter

Datatype

Version

Description

username

String

3.8.2 and later

The user's username.

mbox

String

3.8.2 and later

The user's email address.

attributes

 

3.8.2 and later

 

 

Attributes of a user may be set or changed by PUTting them to the corresponding "User" type entity.

A user's "username" and email address, represented by the "mbox" property", may also be changed.  Special care should be taken when modifying these values since they also serve as keys that distinguish this user from other users.

This action is closely related to POSTing a "User" type value to a Tenant (see Posts to Tenants).  In the case of posting to a tenant, the "username" or "mbox" values are first used to find an existing user.  If found, the remaining parameters are treated exactly as if they were PUT to that user.  If not found, a new user is created with the given "username" and "mbox", and again the remaining parameters are treated as if they were PUT.  The only think you cannot do using the POST to Tenant variant is change a user's username or email address.

Super-admins may PUT to any user entity.  Otherwise, a user may only PUT to their own user entity.
