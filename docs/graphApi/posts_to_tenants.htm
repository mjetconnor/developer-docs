Posts to Tenant Entities

Post requests are handled as defined here when the target is an entity of type Tenant.  The appropriate response is determined by the action and type post properties.

Post a user

Version 3.8.2 and later

Post property

Datatype

Req

Description

action

String

MAY

"post" or undefined

type

String

MUST

"User"

username

String

 

The user's username.

mbox

String

 

The user's email address.

user attribute properties

String

MAY

Other user attributes to set for the user.  They all have String type values.

Returns:  The created or found User entity.

Post a user to the tenant.

A user with the matching username or email address (mbox) is looked up.  If such a user already exists, this post request updates that user's attributes with the values given in this request as if they were PUT to that user (see Puts to Users).  Attributes not included in the request are not changed in the existing user.

If no such user is found, a new user is created using the given username and email address.  All of the additional attribute values given in the request are also set on the new user.

An existing user can be found with either the username or the email address.  Both are not required.  But if a user does not already exist, both are required.  For instance, a request can be made specifying the mbox but not the username.  But if there is no existing user with a matching email address, an error will be returned.

Only a super-admin may execute this request.
